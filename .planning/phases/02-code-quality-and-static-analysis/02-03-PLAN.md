**Part B: cargo-audit (QUAL-06)**

1. Run `cargo audit` on the workspace.
2. If vulnerabilities found:
   - For direct dependencies: Update if patch available
   - For transitive dependencies: Document, assess severity, and update if possible
   - For informational advisories: Document but no action needed
3. Target: zero known vulnerabilities.

**Part C: cargo-deny (QUAL-06)**

1. Create `deny.toml` in workspace root with this configuration:
   ```toml
   [advisories]
   vulnerability = "deny"
   unmaintained = "warn"
   yanked = "warn"
   notice = "warn"

   [licenses]
   unlicensed = "deny"
   allow = [
       "MIT",
       "Apache-2.0",
       "BSD-2-Clause",
       "BSD-3-Clause",
       "ISC",
       "Unicode-3.0",
       "Unicode-DFS-2016",
       "Zlib",
       "CC0-1.0",
   ]
   # Zlib and CC0-1.0 included per review feedback (common in Rust deps)
   # Add more licenses as needed based on first run output
   confidence-threshold = 0.8

   [bans]
   multiple-versions = "warn"
   wildcards = "deny"

   [sources]
   unknown-registry = "deny"
   unknown-git = "allow"  # We use OpenZeppelin git deps
   allow-registry = ["https://github.com/rust-lang/crates.io-index"]
   allow-git = ["https://github.com/OpenZeppelin/stellar-contracts"]
   ```

2. Run `cargo deny check 2>&1`.
3. If license violations found: Add the license to the allow list if it's a permissive open-source license (MIT-like). If it's a copyleft license (GPL), flag for user review. Add a comment in deny.toml for each suppression explaining why it was added.
4. If advisory violations found: Same approach as cargo-audit.
5. Iterate until `cargo deny check` passes (or only has acceptable warnings).

**Part D: Scout Soroban (QUAL-04)**

1. Run `cargo scout-audit` on each in-scope crate:
   ```bash
   cargo scout-audit --manifest-path trading/Cargo.toml 2>&1
   cargo scout-audit --manifest-path strategy-vault/Cargo.toml 2>&1
   cargo scout-audit --manifest-path factory/Cargo.toml 2>&1
   cargo scout-audit --manifest-path price-verifier/Cargo.toml 2>&1
   cargo scout-audit --manifest-path timelock/Cargo.toml 2>&1
   ```

2. For each finding, categorize:
   - **Critical/High genuine**: Fix the code
   - **False positive on fixed-point math**: Add `#[allow(scout_soroban::divide_before_multiply)]` (or similar) with a comment explaining why it's safe. Example: `// Fixed-point: deliberate order for precision via SorobanFixedPoint trait`
   - **Informational/Low**: Document but no code change needed

3. Expected false positives (from research):
   - `divide-before-multiply` in position.rs, market.rs, rates.rs (fixed-point operations)
   - `dos-unbounded-operation` in adl.rs (bounded by MAX_ENTRIES)
   - `iterators-over-indexing` in adl.rs (Soroban Vec limitation)

4. Track all suppressions in a comment block at the top of the affected file or inline, with reasoning per D-13.

5. If Scout cannot be installed (Rust version incompatibility), document the issue clearly in the SUMMARY. Recommend retrying with `rustup override set <compatible-version>` or filing an issue with CoinFabrik.
