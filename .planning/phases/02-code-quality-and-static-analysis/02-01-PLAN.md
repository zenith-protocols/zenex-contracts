3. **timelock/src/errors.rs**: Define `TimelockError` contracterror enum:
   - `NotQueued = 1` (queue entry not found or expired)
   - `NotUnlocked = 2` (timelock delay not yet passed)
   - `Unauthorized = 3` (caller is not owner)
   - `InvalidDelay = 4` (delay value is zero or unreasonable)

4. **timelock/src/events.rs**: Define event structs for audit trail visibility (addresses review concern: auditors/indexers need visibility into timelock operations). Follow the same pattern used in trading/src/events.rs (contracttype struct with `publish` method):
   - `Queued { nonce: u32, target: Address, fn_name: Symbol, unlock_time: u64 }` -- emitted on queue()
   - `Executed { nonce: u32, target: Address, fn_name: Symbol }` -- emitted on execute()
   - `Cancelled { nonce: u32 }` -- emitted on cancel()
   - `StatusSet { target: Address, status: u32 }` -- emitted on set_status()
   - `DelaySet { old_delay: u64, new_delay: u64 }` -- emitted on set_delay()
   Each struct implements `pub fn publish(&self, e: &Env)` using `e.events().publish(("timelock", <action>), self.clone())`.

5. **timelock/src/storage.rs**: Define storage types and helpers:
   - `TimelockKey` contracttype enum: `Delay`, `Nonce`, `Queued(u32)` (u32 is the nonce)
   - `QueuedCall` contracttype struct: `target: Address`, `fn_name: Symbol`, `args: Vec<Val>`, `unlock_time: u64`
   - TTL constants: `ONE_DAY_LEDGERS = 17280`, `LEDGER_THRESHOLD_TEMP = ONE_DAY_LEDGERS * 100`, `LEDGER_BUMP_TEMP = LEDGER_THRESHOLD_TEMP + 20 * ONE_DAY_LEDGERS` (same as governance)
   - Getters: `get_delay(e) -> u64` from instance storage using `unwrap_optimized()`, `get_queued(e, nonce) -> Option<QueuedCall>` from temporary storage (returns None if expired)
   - Setters: `set_delay(e, delay)`, `set_queued(e, nonce, queued)` with TTL extension
   - `next_nonce(e) -> u32`: increment counter in instance storage (same pattern as governance's `next_market_nonce`)
   - `remove_queued(e, nonce)`: remove from temporary storage
   - Import `unwrap::UnwrapOptimized` from soroban_sdk for storage reads (per D-12)

6. **timelock/src/lib.rs**: Implement the generic timelock contract:
   - `#![no_std]`
   - Import: soroban_sdk (contract, contractimpl, contractclient, contracttype, panic_with_error, Address, Env, Symbol, Val, Vec, IntoVal), stellar_access::ownable, stellar_contract_utils::upgradeable::UpgradeableInternal, stellar_macros::{only_owner, Upgradeable}
   - `mod storage; mod errors; mod events;`
   - `#[derive(Upgradeable)] #[contract] pub struct TimelockContract;`
   - Define trait `Timelock` with `#[contractclient(name = "TimelockClient")]`:
     - `fn queue(e: Env, target: Address, fn_name: Symbol, args: Vec<Val>) -> u32;` (owner only)
     - `fn cancel(e: Env, nonce: u32);` (owner only)
     - `fn execute(e: Env, nonce: u32);` (permissionless -- anyone can execute after delay)
     - `fn set_status(e: Env, target: Address, status: u32);` (owner only, instant bypass per D-03)
     - `fn set_delay(e: Env, new_delay: u64);` (owner only, queues via timelock -- see below)
     - `fn get_delay(e: Env) -> u64;`
     - `fn get_queued(e: Env, nonce: u32) -> QueuedCall;`
   - Constructor: `__constructor(e: Env, owner: Address, delay: u64)` -- sets owner via ownable::set_owner, stores delay in instance storage. No trading address stored (per D-02, multi-target).
   - `queue`: `#[only_owner]`, get delay, compute unlock_time = ledger().timestamp() + delay, create QueuedCall, get next nonce, store in temporary storage with TTL, emit `Queued` event, return nonce.
   - `cancel`: `#[only_owner]`, check temporary storage has the key, if not panic NotQueued, emit `Cancelled` event, remove it.
   - `execute`: **CRITICAL -- CEI pattern (addresses review concern: re-entrancy ordering).** Permissionless. Load QueuedCall from temporary storage (unwrap_or_else panic NotQueued). Check unlock_time <= ledger().timestamp() (else panic NotUnlocked). **Remove queue entry FIRST** via `storage::remove_queued(e, nonce)`, **THEN** call `e.invoke_contract::<Val>(&queued.target, &queued.fn_name, queued.args)`. Add comment: `// CEI: remove state before external call. Soroban prevents re-entrancy at the host level, but following CEI is defense-in-depth for auditor confidence.` Emit `Executed` event after invoke returns.
   - `set_status`: `#[only_owner]`, instant bypass. Construct args: `vec![&e, status.into_val(&e)]`. Call `e.invoke_contract::<Val>(&target, &Symbol::new(&e, "set_status"), args)`. Emit `StatusSet` event. (Per D-03)
   - `set_delay`: `#[only_owner]`. This function updates the delay **through the timelock itself** to prevent instant delay reduction attacks. Implementation: queue a call to the timelock's own address with fn_name `"_apply_delay"` and args containing `new_delay`. This means delay changes go through the existing queue/execute flow. Add a separate internal function `_apply_delay(e: Env, new_delay: u64)` that is only callable by the contract itself (`e.current_contract_address().require_auth()`), reads old delay, sets new delay, and emits `DelaySet` event. (Addresses review concern: no way to update delay after deployment without redeploying.)
   - `get_delay`: Return from instance storage.
   - `get_queued`: Load from temporary storage, panic NotQueued if missing.
   - Implement `Ownable` trait: `#[contractimpl(contracttrait)] impl Ownable for TimelockContract {}`
   - Implement `UpgradeableInternal`: check operator auth and owner match (same pattern as governance).
