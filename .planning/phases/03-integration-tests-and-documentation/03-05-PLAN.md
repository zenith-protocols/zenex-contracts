**Step 1: Define coverage measurement policy** (addresses HIGH review concern about undefined measurement):

The coverage policy for this project is:
- **What counts:** Source code in these crates: trading, strategy-vault, factory, price-verifier, governance
- **What's excluded:** All files matching `testutils`, `test.rs`, `test_*` patterns (test infrastructure, not production code)
- **How it's measured:** `cargo llvm-cov --workspace --exclude test-suites` which runs ALL tests in the workspace (both per-crate unit tests and test-suites integration tests) and measures coverage of production code
- **WASM boundary:** Integration tests call contract code via Soroban clients which execute the SAME Rust code (in native mode, not WASM). The cargo-llvm-cov tool instruments native execution, so cross-contract calls from test-suites DO count toward the called contract's coverage.
- **Per-contract check:** After workspace measurement, run per-crate to identify which specific crate has gaps
- **Threshold:** 80% overall workspace coverage (excluding test code). Per-crate breakdowns are informational -- if one crate is at 75% but overall is at 85%, it's acceptable. If one crate is drastically low (<60%), add targeted tests.

# Coverage measurement per-contract (informational breakdown)
coverage-per-crate:
	@echo "=== Workspace Coverage (authoritative measurement) ==="

	@echo ""
	@echo "=== Per-Crate Breakdown (informational) ==="
	@echo "--- Trading ---"
	cargo llvm-cov --package trading --ignore-filename-regex '(testutils|test\.rs|test_)' 2>&1 | tail -3
	@echo "--- Strategy Vault ---"
	cargo llvm-cov --package strategy-vault --ignore-filename-regex '(testutils|test\.rs|test_)' 2>&1 | tail -3
	@echo "--- Factory ---"
	cargo llvm-cov --package factory --ignore-filename-regex '(testutils|test\.rs|test_)' 2>&1 | tail -3
	@echo "--- Price Verifier ---"
	cargo llvm-cov --package price-verifier --ignore-filename-regex '(testutils|test\.rs|test_)' 2>&1 | tail -3
	@echo "--- Governance ---"
	cargo llvm-cov --package governance --ignore-filename-regex '(testutils|test\.rs|test_)' 2>&1 | tail -3

# Mutation testing (best effort, long-running)
